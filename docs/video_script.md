# TaxoConf Video Script

**Estimated duration: 8-10 minutes**

---

## Scene 1: Introduction (0:00 - 0:45)

**[Screen: Overview page with TaxoConf branding visible]**

> Welcome to TaxoConf, a taxonomy-driven framework for conference organization. TaxoConf constructs a hierarchical taxonomy over accepted papers using large language models, then leverages this structure to automate key organizational tasks.
>
> The system supports four functions: paper-reviewer assignment, PC member discovery, oral session organization, and poster session organization. In this video, we'll walk through the main workflow — from configuring the LLM provider to organizing sessions, editing metadata, and exporting the final schedule.

**[Action: Briefly scroll down to show the Input Data section with CSV/JSON tabs, output format cards, and resource links]**

> TaxoConf accepts paper metadata in both CSV and JSON formats. The required fields are ID, title, and authors. An optional abstract field improves topic classification. Results can be exported to Excel, CSV, or HTML.

---

## Scene 2: Setting Up the LLM (0:45 - 1:45)

**[Action: Click "Settings" in the sidebar]**

> Before running any organization task, we need to configure the LLM provider. Click Settings in the sidebar.

**[Screen: Settings page showing LLM Provider & Model panel]**

> Here you can select your LLM provider — OpenAI, Google Gemini, Anthropic Claude, or xAI Grok. The model dropdown automatically updates based on the selected provider.

**[Action: Select a provider, show model dropdown updating]**

> If you have an API key set as an environment variable, it will be detected automatically — you'll see "Key configured" in green next to the provider label. Alternatively, select "Enter manually" and paste your API key directly.

**[Action: Click "Test Connection"]**

> Click Test Connection to verify that the API key and model are working. You should see a success message confirming the model responded.

**[Action: Click "Save Settings"]**

> Click Save Settings to apply your changes. The LLM is now ready to power the session organization.

---

## Scene 3: Oral Session Organization (1:45 - 5:30)

### 3.1 Loading Data and Configuring Parameters (1:45 - 2:30)

**[Action: Click "Oral Session Organization" in the sidebar]**

> Now let's organize oral sessions. Click Oral Session Organization in the sidebar. The workspace selector at the top shows our conference data — in this case, SIGIR25 with 251 papers.

**[Screen: Oral setup panel with toolbar and 3-column configuration]**

> The setup panel shows three areas. On the left, Session Parameters: the number of parallel sessions M, time slots N, and the minimum and maximum papers per session. These values are automatically computed from the paper count — but you can adjust them manually. There's also a checkbox to avoid presenter conflicts across parallel sessions.
>
> In the center, the Capacity Check confirms whether the current configuration can accommodate all papers. On the right, you can select the export format.

### 3.2 Running the Organization (2:30 - 3:15)

**[Action: Click "Run Oral Organization"]**

> Click Run Oral Organization. The system sends the papers to the LLM, which builds a taxonomy and groups papers into sessions. This typically takes 30 seconds to a few minutes depending on the paper count and LLM speed.

**[Screen: Loading state, then results appear. Setup panel auto-collapses, sidebar collapses.]**

> Once complete, the setup panel collapses automatically to give the results maximum screen space. A summary chip shows the key metrics — conference name, paper count, number of sessions, and the grid dimensions.

### 3.3 Reviewing the Schedule Grid (3:15 - 4:00)

**[Screen: Schedule grid with session tiles, slot metadata, track headers]**

> The result is a two-dimensional schedule grid. Each row is a time slot, each column is a location. The track names — Location A, Location B, and so on — are editable. Simply type a new name in the header and it applies to all sessions in that column.
>
> Each time slot has inline date and time pickers on the left. Setting the date and time for one slot automatically propagates to all parallel sessions in that slot.
>
> Each session tile shows the session name generated by the LLM, the paper count, and a badge indicating whether the session is within capacity bounds.

### 3.4 Editing Session Metadata (4:00 - 4:45)

**[Action: Click a session tile to open the slide-in detail panel]**

> Click any session tile to open the detail panel. Here you can edit the session name, assign a session chair, set the date, start time, end time, and room location. Time and date changes apply to all parallel sessions in the same slot — a note at the top reminds you of this.
>
> Below the metadata, you'll see the list of papers assigned to this session. You can move any paper to a different session by selecting a target from the dropdown and clicking Move.

**[Action: Edit a field, then click "Save Session Metadata"]**

> After editing, click Save Session Metadata. Changes are reflected immediately in the schedule grid.

### 3.5 Last-Mile Modifications (4:45 - 5:15)

**[Screen: Scroll down to the Last-Mile Modification panel]**

> Below the schedule grid is the Last-Mile Modification panel. The LLM reviews all sessions and flags papers that may be poorly placed — for example, a paper on brain image reconstruction placed in a retrieval-augmented generation session.
>
> Click a flagged paper to see the reason it was flagged and a list of alternative sessions. Select a better-fit session and click Apply to reassign it.

### 3.6 Saving, Loading, and Exporting (5:15 - 5:30)

**[Action: Click "Save Progress" in the toolbar]**

> Click Save Progress to open the save dialog. Enter a name — for example, "oral draft 1" — and click Save. Your edits are preserved on the server, and you can return to them later.

**[Action: Click "Load Progress"]**

> Click Load Progress to see a list of all saved versions with timestamps. Click any save to restore it.

**[Action: Click "Export" with Excel selected]**

> To export, select your format — Excel, CSV, or HTML — and click Export. The Excel export uses a conference template and fills in the date, time, track, session title, room, and session chair columns.

---

## Scene 4: Poster Session Organization (5:30 - 7:30)

### 4.1 Configuration (5:30 - 6:00)

**[Action: Click "Poster Session Organization" in the sidebar]**

> Poster session organization follows a similar workflow. Click Poster Session Organization. The setup panel shows floor plan options — you can choose a Rectangle layout with rows and columns, or a Line or Circle layout with a board count.
>
> Set the session count and optionally enable the checkbox to prevent the same presenter from appearing in the same session.

### 4.2 Running and Reviewing (6:00 - 6:45)

**[Action: Click "Run Poster Organization"]**

> Click Run Poster Organization. After the LLM processes the papers, the results appear as a grid of poster session cards. Each card shows the session name, paper count, and the floor plan layout with individual board positions.

**[Screen: Poster session grid with session cards and board layouts]**

> Click any session card to open the detail panel. The metadata editing is consistent with oral sessions — session name, chair, date, time, and room, all using native date and time pickers. No description or speaker fields to worry about.

### 4.3 Editing, Saving, and Exporting (6:45 - 7:30)

**[Action: Show session detail panel, edit metadata, save]**

> You can move papers between sessions, reassign boards, and edit all metadata just like in oral organization. The last-mile modification panel works the same way — flagged papers with suggested alternatives.

**[Action: Click "Save Progress", then "Export"]**

> Save your progress, load previous saves, and export to Excel, CSV, or HTML — the same workflow as oral sessions. The exported Excel follows the same conference template schema.

---

## Scene 5: Token & Cost Tracking (7:30 - 8:00)

**[Action: Click "Token & Cost" in the sidebar]**

> Finally, let's check our LLM usage. Click Token & Cost in the sidebar.

**[Screen: Token stats page with three summary cards and run history table]**

> The page shows three summary cards. The Last Run card shows prompt tokens, completion tokens, total tokens, and estimated cost for the most recent operation. The Current Workspace card aggregates usage across all runs in this workspace. The Global card shows usage across all workspaces.
>
> Below is the Run History table listing every LLM call with its mode, timestamp, token counts, cost, provider, and model. You can reset workspace or global counters using the Reset buttons.

> That's TaxoConf — a complete workflow for organizing conference oral and poster sessions using LLM-powered taxonomy construction. From data input to session scheduling, metadata editing, and export, everything happens in one integrated workspace.
